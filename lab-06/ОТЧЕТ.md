### 4. Анализ выразительности кода

#### Задача: Фильтрация + суммирование

**Haskell** (самый лаконичный):
```haskell
sum [item * qty | (item, qty) <- products, qty > 0]
```

**Python** (очень читаемый):
```python
sum(item * qty for item, qty in products if qty > 0)
```

**Scala** (компактный):
```scala
products.filter(_._2 > 0).map(p => p._1 * p._2).sum
```

**Rust** (многословный, но явный):
```rust
products.iter()
    .filter(|(_, qty)| *qty > 0)
    .map(|(item, qty)| item * qty)
    .sum()
```

**JavaScript** (читаемый, но длинный):
```javascript
products.filter(p => p.qty > 0)
    .map(p => p.item * p.qty)
    .reduce((a, b) => a + b, 0)
```

### 5. Обработка ошибок: сравнение подходов

#### Haskell (Maybe/Either)
```haskell
findOrder :: Int -> Maybe Order
findOrder id = lookup id orderMap

case findOrder 42 of
    Just order -> processOrder order
    Nothing -> putStrLn "Заказ не найден"
```

**Преимущества:** Ошибка явно выражена в типе  
**Недостатки:** Требует обработки во всех местах

#### Scala (Try/Option)
```scala
val order = orderMap.get(42)
order match {
    case Some(order) => processOrder(order)
    case None => println("Заказ не найден")
}
```

**Преимущества:** Явное выражение возможности ошибки  
**Недостатки:** Больше кода

#### Rust (Result)
```rust
fn find_order(id: u32) -> Result<Order, String> {
    order_map.get(&id)
        .ok_or("Заказ не найден".to_string())
}

match find_order(42) {
    Ok(order) => process_order(order),
    Err(e) => println!("Ошибка: {}", e),
}
```

**Преимущества:** Безопасно, явно, с информацией об ошибке  
**Недостатки:** Требует явной обработки

#### Python (Исключения)
```python
try:
    order = find_order(42)
    process_order(order)
except OrderNotFoundError as e:
    print(f"Ошибка: {e}")
```

**Преимущества:** Скоро писать  
**Недостатки:** Ошибка скрывается, сложно отследить

#### JavaScript (Исключения)
```javascript
try {
    const order = findOrder(42);
    processOrder(order);
} catch (e) {
    console.error(`Ошибка: ${e.message}`);
}
```

**Преимущества:** Простой синтаксис  
**Недостатки:** Нет гарантии обработки, скрытые потоки ошибок

## Ответы на контрольные вопросы

### 1. Какой язык оказался наиболее выразительным для ФП и почему?

**Ответ:** Haskell и Scala одинаково выразительны (9/10 каждый).

**Почему:**
- **Haskell:** Синтаксис максимально соответствует математической нотации. List comprehensions, pattern matching, type classes.
- **Scala:** Case classes, for-comprehensions, вывод типов делают код лаконичным.

**Для практического применения Scala выигрывает** благодаря JVM экосистеме и лучшей интеграции с существующими инструментами.

---

### 2. Какие компромиссы между безопасностью типов и продуктивностью разработки вы заметили?

**Ответ:** Четкий спектр компромиссов:

| Типизация | Безопасность | Скорость разработки | Язык |
|-----------|-------------|-------------------|------|
| Максимальная | 10/10 | 3/10 | Haskell, Rust |
| Высокая | 9/10 | 5/10 | Scala |
| Низкая | 4/10 | 8/10 | Python |
| Минимальная | 3/10 | 9/10 | JavaScript |

**Вывод:** Инвестиция в типизацию экономит время на отладку в долгосрочной перспективе.

---

### 3. Как разные языки решают проблему побочных эффектов?

**Ответ:**

- **Haskell:** Изолирует эффекты в Monad IO (безопасно)
  ```haskell
  main :: IO ()  -- явно указано, что есть эффекты
  ```

- **Scala:** Использует for-comprehensions и Try/Either (практично)
  ```scala
  for {
    user <- findUser(id)  // может вернуть None/error
    order <- processOrder(user)
  } yield order
  ```

- **Rust:** Result/Option types (гарантирует обработку)
  ```rust
  fn process() -> Result<(), Error> { }  // обработка обязательна
  ```

- **Python/JS:** Соглашения и документация (нет гарантий)
  ```python
  def process():  # может быть исключение, может не быть
      pass
  ```

**Вывод:** ФП языки явно выражают эффекты в типах, императивные — нет.

### 4. Какой язык вы бы выбрали для high-performance приложения и почему?

**Ответ:** **Rust** — уникальная комбинация факторов:

1. **Максимальная производительность** (нулевые абстракции, нет GC)
2. **Безопасность памяти** (ownership система, compile-time гарантии)
3. **Отсутствие runtime ошибок** (Result/Option типы)
4. **Параллелизм без race conditions** (жесткие правила компилятора)

**Альтернативы:**
- **Scala + Spark:** Для распределенных вычислений
- **Haskell:** Для алгоритмически сложных задач (корректность важнее скорости)

### 5. Какие особенности Rust делают его уникальным среди других языков?

**Ответ:** 5 уникальных особенностей:

1. **Система владения (ownership)**
   - Гарантирует безопасность памяти без GC
   - Компилятор отслеживает жизненный цикл каждого значения
   - Ошибка: use-after-free → compile error (не runtime error)

2. **Заимствование (borrowing)**
   - Можно передавать ссылки без передачи владения
   - Без runtime накладных расходов
   - `&T` (иммутабельная) vs `&mut T` (мутабельная)

3. **Нулевая стоимость абстракций (Zero-cost abstractions)**
   - Высокоуровневые конструкции компилируются в машинный код без потерь
   - Iterator chains не добавляют runtime overhead

4. **Гарантии параллелизма (Send + Sync traits)**
   - Компилятор предотвращает race conditions
   - Ошибка: отправка non-Send типа в другой поток → compile error

5. **Отсутствие исключений (результат + Option)**
   - Нет surprise panics
   - Явно выражена возможность ошибки в типе
   - Обязательная обработка или явное распространение

## Заключение

### Главные открытия

1. **Нет "лучшего" языка** — есть оптимальные языки для разных задач

2. **Спектр компромиссов:**
   - Выразительность ↔ Производительность
   - Безопасность типов ↔ Скорость разработки
   - Простота ↔ Мощь

3. **Функциональные парадигмы улучшают код:**
   - Иммутабельность → проще рассуждать
   - Чистые функции → проще тестировать
   - Типизированные ошибки → меньше bugs

4. **Тренд: гибридные языки**
   - Scala (ООП + ФП)
   - Kotlin (ООП + ФП на JVM)
   - TypeScript (динамика + статическая типизация)
   - Rust (системное + функциональное)

### Рекомендация

**Изучите несколько языков:**
- **Python** - для понимания ФП концепций
- **JavaScript** - для практического применения в вебе
- **Scala** или **Haskell** - для углубленного понимания типов
- **Rust** - для системного программирования и безопасности
